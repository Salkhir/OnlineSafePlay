Accessibility statement

SafeOnline UK is committed to making this website accessible to everyone — including people with disabilities. This statement was last reviewed in June 2026.

Our target: WCAG 2.1 Level AA

We aim to meet the Web Content Accessibility Guidelines (WCAG) 2.1 at Level AA — the standard recommended by the UK Government and the Equality Act 2010. We are working towards full compliance across all pages.

What we have done to make this site accessible

  • All pages include a visible skip-to-content link for keyboard users
  • All images and icons that convey meaning include alt text or aria-label attributes
  • Decorative images and icons are marked with aria-hidden="true"
  • All interactive elements are keyboard operable and display a visible focus indicator
  • Colour contrast ratios meet WCAG 2.1 AA minimum requirements (4.5:1 for normal text, 3:1 for large text)
  • Font sizes use relative units (px based on user browser defaults) and scale when browser text size is increased
  • The site uses semantic HTML elements (nav, main, header, footer, section, article)
  • Form inputs have associated labels and error messages are descriptive
  • Interactive tabs, accordions, and quiz elements use appropriate ARIA roles and attributes
  • Animated elements respect the prefers-reduced-motion media query
  • The site works without JavaScript for all core content — interactive tools degrade gracefully
  • All pages include a language attribute (lang="en")
  • Page titles are descriptive and unique
  • Link text is descriptive in context — we avoid "click here" and "read more"

Known limitations

Despite our best efforts, some areas of the site have known accessibility limitations. We are actively working to address these:

AreaIssueOur plan
Family tech contract builder The live preview panel may not announce updates to screen reader users in real time We will review and improve ARIA live region implementation in a future update
Platform settings hub Tab switching may not announce the new panel title to all screen readers We will improve the tab ARIA pattern to announce panel content on switch
Downloadable resources PDFs linked from the resources page are provided by third parties and may not be fully accessible We will flag inaccessible PDFs and offer HTML alternatives where possible

Compatibility

This website is designed to be compatible with:

  • Recent versions of Chrome, Firefox, Safari, and Edge on desktop
  • Mobile browsers on iOS (Safari) and Android (Chrome)
  • Screen readers including NVDA, JAWS, and VoiceOver
  • Browser zoom up to 400% without loss of content or functionality
  • High contrast modes on Windows and macOS

The site does not support Internet Explorer. If you are using IE, we recommend switching to a modern browser for the best experience.

Technical information

SafeOnline UK is built using HTML5, CSS3, and vanilla JavaScript. We do not use frontend frameworks that may introduce accessibility barriers. Icon fonts are used from Tabler Icons and are marked decorative where they duplicate adjacent text.

We use ARIA attributes where native HTML semantics are insufficient, and only where they are supported by the assistive technologies listed above.

Requesting accessible content

If you need any content on this site in an alternative format — such as large print, easy-read, or audio description — please contact us and we will do our best to help. We aim to respond to accessibility requests within 5 working days.

Reporting accessibility problems

We welcome feedback on the accessibility of this site. If you find a barrier that prevents you from accessing content, please report it to us with as much detail as possible — including the page URL, the assistive technology you are using, and what you were trying to do. We will investigate and respond within 5 working days.

Enforcement

If you are not satisfied with our response to an accessibility issue, you can contact the Equality Advisory and Support Service (EASS), which provides advice and support with discrimination and human rights issues.

The Equality Act 2010 requires public sector bodies and organisations providing services to the public to make reasonable adjustments to remove barriers for disabled users.

How we prepared this statement

This statement was prepared in June 2026 following a self-assessment of all pages against WCAG 2.1 Level AA criteria. We plan to commission an external accessibility audit in the next 12 months and will update this statement with the findings.
